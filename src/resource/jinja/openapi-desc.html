Header 예시
- Authorization : bearer 1234567890...
- Content-Type : application/json
Response 구조
- 성공 = { "리소스명": dict, ... }
- 실패 = { "err_cd": str, "err_msg": str }
StatusCode
- 200 = 성공 (OK)
- 400 = 잘못된 요청오류1 (Bad Request)
- 401 = 인가오류 (Unauthorized)
- 403 = 사용자의 권한오류 (Forbidden)
- 404 = 요청한 리소스가 없음 (Not Found)
- 422 = 잘못된 요청오류2 (Unprocessable Entity)
- 500 = 서버 장애 (Internal Server Error)
- 503 = 서버 과부하 (Service Unavailable)
- 555 = 서버 DB Exception (커스텀)
- 999 = 서버 Base Exception (커스텀) - ...
